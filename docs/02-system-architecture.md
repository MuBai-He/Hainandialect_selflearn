# 系统架构文档

## 1. 架构目标

系统架构设计需要服务于当前 Web 第一版的快速落地，同时为后续接入 AI 服务、后台管理、多端扩展预留空间。

当前版本的设计原则是：

- 先单体落地，后服务拆分
- 先业务闭环，后智能增强
- 先内容驱动，后复杂模型驱动

## 2. 当前版本推荐架构

Web 第一版建议采用轻量单体架构：

```text
客户端层
├─ Vue Web 前端

服务层
├─ Spring Boot API

数据层
├─ MySQL
```

说明：

- 第一版不强制拆分 AI 服务
- 第一版不强制引入 Redis
- 第一版不强制引入对象存储
- 翻译逻辑先放在 Spring Boot 内部实现

这样做的优势：

- 架构简单
- 调试和联调成本低
- 更适合当前阶段快速交付

## 3. 后续扩展架构

当 Web 第一版稳定后，可扩展为如下结构：

```text
客户端层
├─ Web 用户端
├─ Admin 管理端
└─ 小程序端

服务层
├─ Spring Boot 业务 API
├─ FastAPI AI 服务
└─ 定时任务或异步任务服务

数据层
├─ MySQL
├─ Redis
└─ 对象存储
```

## 4. 技术栈说明

## 4.1 前端

- `Vue 3`
- `Vite`
- `Vue Router`
- `Pinia`
- `Axios`

说明：

- 当前 `apps/web` 已经是 Vue + Vite 基础骨架
- 第一版建议先使用轻量自定义组件方案
- 后续如果页面和表单复杂度上升，再引入 UI 组件库

## 4.2 后端

- `Spring Boot 3`
- `Spring Web`
- `Spring Validation`
- `Spring Data JPA`
- `BCrypt`
- `JWT`

说明：

- 第一版主要负责认证、学习内容、翻译、收藏、学习记录等业务逻辑
- 当前认证方案采用 `密码登录 + JWT Bearer Token`

## 4.3 数据库

- `MySQL 8`

用途：

- 存储用户信息
- 存储学习内容
- 存储翻译词库
- 存储学习记录与翻译历史

## 4.4 后续可选组件

### Redis

适用时机：

- 接口访问量上升
- 需要缓存高频翻译词库
- 需要增强登录态管理

### 对象存储

适用时机：

- 接入音频、图片、用户上传文件

### FastAPI AI 服务

适用时机：

- 接入大模型翻译
- 接入发音评分
- 接入推荐系统

## 5. 分层职责

## 5.1 Vue Web 前端

职责：

- 页面展示
- 用户交互
- 登录态维护
- 调用后端接口
- 页面路由与状态管理

## 5.2 Spring Boot API

职责：

- 用户注册登录
- 用户信息管理
- 首页聚合数据返回
- 学习模块与课程数据返回
- 学习完成状态记录
- 翻译请求处理
- 收藏与历史记录管理

## 5.3 MySQL

职责：

- 结构化业务数据持久化
- 用户行为数据记录
- 内容数据管理

## 6. 核心业务流程

## 6.1 登录流程

```text
前端提交登录表单
-> Spring Boot 校验用户信息
-> 返回 Token
-> 前端保存登录态
-> 后续请求带上认证信息
```

## 6.2 学习流程

```text
用户进入学习中心
-> 请求模块列表
-> 打开课程详情
-> 查看词汇和短句
-> 完成课程
-> 后端记录学习状态
-> 个人中心展示学习进度
```

## 6.3 翻译流程

```text
用户输入文本
-> 前端提交翻译请求
-> 后端进行词库匹配和规则处理
-> 返回翻译结果
-> 后端记录翻译历史
-> 用户可收藏结果
```

## 6.4 收藏流程

```text
用户在课程或翻译页点击收藏
-> 前端调用收藏接口
-> 后端写入收藏记录
-> 个人中心返回收藏列表
```

## 7. 目录与模块建议

结合现有仓库，建议保持以下职责分布：

```text
apps
├─ web         # 用户端 Web
├─ api         # Spring Boot 业务接口
├─ admin       # 后续后台
├─ ai-service  # 后续 AI 服务
└─ miniapp     # 后续小程序
```

当前开发重点：

- `apps/web`
- `apps/api`
- `database`
- `docs`

当前可不投入开发：

- `apps/admin`
- `apps/ai-service`
- `apps/miniapp`

## 8. 部署建议

## 8.1 第一版部署

推荐部署方式：

- 前端静态打包部署
- Spring Boot 单实例部署
- MySQL 单实例部署

如果本地开发：

- 前端使用 Vite 开发服务器
- 后端使用 `spring-boot:run`
- 数据库使用本地 MySQL 或 Docker

## 8.2 后续扩展部署

扩展方向：

- 增加 Nginx 反向代理
- 引入 Docker Compose
- 引入 Redis
- 拆分 AI 服务

## 9. 关键设计决策

- 当前阶段采用单体业务 API，降低开发复杂度
- 将 AI 能力视为后续增强，而不是第一版依赖项
- 所有核心能力先围绕学习闭环展开
- 架构设计优先服务当前交付，而不是为了未来过度设计
