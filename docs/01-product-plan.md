# 产品规划文档

## 1. 产品定位

海南方言智能传播与学习平台是一个面向海南方言学习、查询与传播的数字化产品。

当前阶段的核心目标不是一次性做成完整的 AI 语言平台，而是先完成一个可上线、可演示、可验证需求的 Web 第一版，建立最基础的用户学习闭环。

## 2. 当前版本目标

Web 第一版聚焦五个核心页面：

- 首页
- 注册登录
- 学习中心
- 基础翻译
- 个人中心

核心价值主张：

- 让用户能快速了解平台
- 让用户能开始学习海南方言
- 让用户能完成基础短句翻译
- 让用户能看到自己的学习记录和收藏内容

## 3. 目标用户

### 3.1 核心用户

- 海南方言零基础学习者
- 海南本地年轻用户
- 新来海南生活或工作的外来人员
- 对海南文化感兴趣的普通用户

### 3.2 后续扩展用户

- 学校与教师群体
- 文旅推广相关人员
- 政企培训场景用户

## 4. 第一版范围

## 4.1 本版必须实现

### 用户模块

- 用户注册
- 用户登录
- 用户退出登录
- 当前用户信息获取

### 首页模块

- 平台介绍
- 核心功能入口
- 推荐课程展示
- 常用短句展示

### 学习模块

- 学习模块列表
- 课程列表
- 课程详情
- 词汇与短句展示
- 学习完成记录

### 翻译模块

- 普通话到海南方言基础翻译
- 海南方言到普通话基础翻译
- 翻译历史记录
- 翻译结果收藏

### 个人中心模块

- 用户资料展示
- 学习进度展示
- 最近学习记录
- 收藏内容展示
- 翻译历史查看

## 4.2 本版不做

- 数字人
- 发音打分
- 实时语音识别
- 三语互译中的英语能力
- 知识图谱可视化
- 社区功能
- 众包语料上传
- 独立管理后台

说明：

这些能力不是没有价值，而是不适合在第一版同时开发。第一版优先保证产品闭环与可交付性。

## 5. 页面与功能关系

```text
首页
├─ 进入学习中心
├─ 进入翻译页
└─ 引导注册登录

注册登录
└─ 建立用户身份与登录态

学习中心
├─ 模块列表
├─ 课程详情
└─ 学习完成记录

基础翻译
├─ 文本输入
├─ 结果返回
└─ 收藏与历史

个人中心
├─ 用户信息
├─ 学习记录
├─ 收藏内容
└─ 翻译历史
```

## 6. 内容建设原则

第一版不追求内容量大而全，建议先准备一批高频、强场景、易演示的种子内容。

优先内容方向：

- 日常问候
- 餐饮点单
- 交通出行
- 常见称呼
- 购物支付

建议初始内容规模：

- 5 个学习模块
- 每个模块 4 到 6 节课程
- 150 到 250 个高频词汇
- 80 到 120 条常用短句
- 100 到 200 条翻译词库

## 7. 版本规划

## 7.1 P0 阶段：产品与骨架准备

目标：

- 明确第一版范围
- 调整项目骨架
- 完成数据库与接口边界设计

交付：

- 产品文档
- 页面结构图
- 数据库表设计
- 接口清单

## 7.2 P1 阶段：Web 第一版 MVP

目标：

- 完成首页、注册登录、学习中心、基础翻译、个人中心

交付：

- Vue 前端可运行版本
- Spring Boot 后端接口
- MySQL 数据库
- 初始化种子内容

## 7.3 P2 阶段：体验增强

目标：

- 提升学习效果和留存效果

方向：

- 更丰富的练习题型
- 学习推荐
- 签到与积分
- 内容运营能力

## 7.4 P3 阶段：AI 增强

目标：

- 在产品稳定后接入更强智能能力

方向：

- AI 翻译增强
- 发音分析
- 数字人互动
- 知识图谱联动

## 8. 核心指标

第一版建议关注以下指标：

- 注册用户数
- 登录转化率
- 课程完成数
- 翻译调用次数
- 收藏使用率
- 7 日留存

## 9. 风险与优化建议

### 风险一：功能范围膨胀

问题：

- 原始设想较大，容易在开发过程中不断追加需求

建议：

- 所有超出五个核心页面的需求统一进入后续版本池

### 风险二：翻译能力预期过高

问题：

- 如果第一版直接追求复杂 AI 效果，容易拖慢开发进度

建议：

- 第一版采用词库、规则、短句模板方案
- 先解决高频短句场景

### 风险三：内容不足导致页面空洞

问题：

- 前端页面做完后，如果没有课程和词条内容，体验会很差

建议：

- 优先准备种子课程和翻译词库
- 内容建设与开发同步推进
