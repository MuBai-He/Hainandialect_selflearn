# 言通琼岛 Web 前端页面地图

## 1. 文档目标

这份页面地图用于统一 Web 端扩展页面的设计、路由和实现边界，确保新增页面在视觉风格、命名方式和信息架构上与现有首页、学习中心、翻译页、个人中心保持一致。

当前扩展页定位分为三层：

- 作品展示层：服务于答辩、评委浏览、项目理解
- 学习能力层：服务于学习闭环、内容深度、AI 展示
- 个人沉淀层：服务于成果呈现、学习复盘、长期留存

## 5. 路由命名约定

- `path` 使用英文小写加中划线或嵌套路径
- `name` 使用语义清晰的 kebab-case
- 列表页保持短路径，便于答辩现场输入与切换
- 详情页统一使用 `:id`、`:entryId`、`:storyId`、`:scenarioKey` 这类可读参数名

示例：

- `/about` -> `about-project`
- `/lab/pronunciation` -> `pronunciation-lab`
- `/dictionary/:entryId` -> `dictionary-entry`
- `/stories/:storyId` -> `story-detail`

## 6. 权限建议

- 公开访问：`/about`、`/archive`、`/showcase`、`/dictionary`、`/roadmap`、`/scenarios/:scenarioKey`、`/stories/:storyId`
- 建议登录后访问：`/report`

## 7. 设计一致性要求

- 统一使用现有青黛、朱砂、米白、金色 token
- 继续保留海浪 SVG 分隔、印章式品牌识别、档案感卡片与深色文化叙事区
- 所有新增页面必须有完整 mock 内容，不出现“0”与“正在加载...”
- 所有主要按钮、卡片和标签继续保持 hover / active / dark mode 适配
